# 📜 Policy for chat.py

歡迎來到 chat.py！本文件概述了參與本專案所需的政策和準則，旨在建立一個高效、協作且友善的社群環境。

## 1. 🤝 行為準則 (Code of Conduct)

我們致力於為所有人提供一個**免受騷擾**的社群體驗，不論其性別、性取向、身心障礙、種族、宗教或技術經驗。

### 1.1 我們的承諾

* **尊重與包容：** 避免人身攻擊、貶低性評論以及其他形式的冒犯性行為。
* **建設性回饋：** 專注於程式碼、設計或技術的討論，而非針對個人。
* **專業性：** 在所有交流中保持禮貌和專業。

### 1.2 報告和處置

如果您遇到或目擊違反本行為準則的行為，請立即聯繫專案維護者：

* **聯繫人/郵箱：** <hray1413incoffical@gmail.com>
* 所有投訴將被審核和調查，並將導致適當的處置，可能包括警告、暫時禁言或永久排除社群。

## 2. 💻 貢獻政策 (Contribution Policy)

本節說明了如何向專案提交程式碼、文檔和建議的準則。

### 2.1 提交問題 (Issue)

在提交任何程式碼之前，請先檢查是否有現存的Issue。

* **錯誤 (Bugs)：** 請提供清晰的**重現步驟**、預期行為和實際行為。
* **功能請求 (Features)：** 請描述功能的用途、目標以及它如何幫助專案。
* **討論：** 對於重大變更，請先在 Issue 中進行討論，以獲得維護者的初步批准。

### 2.2 程式碼標準

我們期望所有貢獻的程式碼符合以下標準：

* **風格指南：** 必須遵循 **PEP 8** Python 風格指南。關鍵要求包括：
    * **行長度：** 單行程式碼長度應限制在 **79 個字元**以內。
    * **命名慣例：** 函式和變數使用 `snake_case`；類別使用 `CapWords`；常數使用全大寫。
    * **縮排：** 必須使用 **4 個空格**進行縮排，不允許使用 Tab 鍵。
    * **空白行：** 遵循 PEP 8 的標準，使用空白行分隔頂層和類別內部結構。
* **類型提示 (Type Hinting)：** 建議為所有新函式和方法的參數及返回值添加 **Python 類型提示**。
* **測試：** 新增的功能必須包含**單元測試**，以確保程式碼的穩定性。
* **文檔：** 任何面向用戶的變更或新功能都應更新**相關文檔**。

### 2.3 提交拉取請求 (Pull Request, PR)

所有程式碼變更都必須透過 PR 流程：

1.  **分支出發：** 始終從最新的 `main` 或 `develop` 分支建立一個**獨立的分支**。
2.  **單一職責原則：** 每個 PR 應盡可能只處理**一個問題或功能**。
3.  **描述：** 在 PR 描述中，請解釋變更的內容、動機，並連結到相關的 Issue 編號 (`Fixes #123`)。
4.  **授權聲明：** 透過提交 PR，您聲明您有權利且願意根據本專案的授權許可您的貢獻。
